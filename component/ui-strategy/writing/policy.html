Title
Popup Title
Use the command name that users selected as the title of the pop-up window.
In English, use the title capitalization and do not use any articles.
Examples of pop-up titles are as follows.
Message
If a message title is a complete sentence, do not use standard title capitalization or a period.
Error Message
Use the “Failed to” or “Unable to” constructs depending on the types of error messages.
Use the "Failed to" construct if there was a system error but users can complete the action by retrying it.
This type of error message should be followed by "Try again?"
Use the "Unable to" construct if it is a temporary problem, such as a server being down, that will be resolved in time.
This type of error messages should be followed by a reason or a solution to the problem.
In Korean, use the verb ending “-할 수 없습니다” rather than “-하지 못했습니다” or “실패했습니다.”
Settings Descriptions (RIS)
For supplemental descriptions in the Settings menus, write sentences starting with a verb in English, and use the verb ending “-합니다” in Korean.
Tooltip Message
Show only command names in tooltip messages.
In English, use standard title capitalization.
Button
Screen(UI) Button

In English, use standard title capitalization and do not use any articles. When naming a button, start with a verb that matches the action that will occur when it is selected.

Do not use any descriptor after an on-screen (UI) button name, such as “button.”

Exception)
Use the descriptor “button” to avoid confusion or awkward phrases. For example, use “Select the Select button to...” not “Select Select to...” if “Select” is the button name.
Exception) Cancel, Exit
Use “Yes” and No” for button names when the primary action is to cancel or exit, because users may confuse whether the job will be canceled or the pop-up window will be closed.
In Korean, enclose on-screen (UI) button names with single quotation marks.
Examples of button names are as follows:
PUI Button
Use the following constructs for pressing physical buttons on a remote control.

- English: Press the xx button.
- Korean: xx 버튼을 누르세요.
If a physical button has a printed label, use the button name as it appears. Printed button labels may differ between the U.S. and Korea, so be sure to check the labels when referring to physical buttons.
Variables
Use “<<A>>” for variables in messages. If there are multiple variables used in one message, use “<<A>>,” “<<B>>,” “<<C>>,” and so on.
Clearly state what elements the variables will be replaced with in the comments.
Comment: <<A>> = button name <<B>> = button name <<C>> = number
Comment: Example item for Voice Recognition. <<A>>=volume number
Use variables for the following items:
- Combinations of digits, such as date and time, or uncertain.
- Marketing names that have not been decided, such as a remote control name.
- Menu path items. Menu paths may change depending on the software version, so use variables to avoid updating the message every time the menu path changes. However, specific menu paths must be clearly stated in the IA document.
<<A>> = path for settings menu (Settings > Design Element )
In Korean, add a descriptor after a variable, if appropriate, to avoid adding both allomorphs of a postposition directly after the variable.
If there is no workaround, use both allomorphs of an appropriate postposition after a variable, one without any grammatical symbols and the other enclosed with parentheses.
Control
For mobile devices, use the verb “tap” for the action of selecting something on the screen.
Use the verb “press” for the action of pressing buttons on the remote control and the verb “push up or down” for the action of pushing up or down the jog button on the remote control.
Use the verb “click” for the action of selecting something on the computer screen with a mouse.
Use the verb “select” for the action of selecting something on the TV screen with a remote control.
Enclose direct speech with double quotation marks.
