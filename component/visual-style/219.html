21:9 Aspect ratio

  • Definition
  • Rule
Definition

When UI elements that are initially made for 16:9 ratio is loaded onto 21:9 wide screen,
those UI elements can also be best viewed at 21:9 ratio through realignment based on situational principles.

Rule
1. Center align

For UI components that are not adjacent to left, right, top, bottom sides, keep the alignment at center.

2. Right side expansion

For UI components that are list type and over one-line, alignment should be to the left with the list expanding to the right.
This is so that more content list can be shown to the user.

3. Position Change

When screen size is adjusted depending on widget's on/off status within 21:9 area,
the UI components that are located on left or right edge are positioned to be proportional to the adjusted screen size

4. Left & Right blank expansion

In order to avoid informational area from being too wide at 21:9 ratio,
maintain the position to be the same as 16:9 and expand only the background area. This is so that no information goes beyond the recognizable area.
