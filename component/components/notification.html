C_NOTIFICATION
Definition
Use a notification to show users event alerts or the system status that are not related to the current task.
A notification should take up the minimal amount of space so as not to interrupt the user experience.
Usage
  • - Use this, like program reminders, to let users know the results of the previous action or push alerts from the connected devices.
  • - Display event notifications at the top of the screen.
Specs
Properties
Notification consists of the following elements:
1. Icon/Image (Mandatory)
  • - display only 1 icon or image. NOT display both icon and imavbge at the same time
2. Main Text (Mandatory)
  • - the message that users should be informed of
3. Sub text (Optional)
  • - the source of the notification : sender, device, app, CH information, etc.
  • - clip any overflowed text and replace it with an ellipsis (...) without auto scrolling
4. Button (Optional)
  • - Maximum number of buttons : 3
  • - Place the recommended button first and put the initial focus on it
  • - **refer to Basic Button for details
5. Progress bar (Optional)
  • - Display a progress bar to show the status of an ongoing task (**refer to Progress Bar for details)
