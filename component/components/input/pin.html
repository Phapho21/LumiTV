C_PIN1LINE
Definition

PIN Fields are special input field when user inputs number as PIN.

Usage
  • - Use a PIN field to allow users to enter their PINs.
  • - Use a one-line PIN field when users must enter their PINs to perform a task.
  • - Use a two-line PIN field when users must enter their PINs two times, such as when changing their PINs.
  • - Provide a Cancel button to allow users to cancel their input and return to the previous screen.
  • - When the PIN field appears in a popup, specify the popup’s title as “Lock” or “Unlock” unless there is a specific title to enhance user’s comprehension.
    If “Lock” or “Unlock” is not proper title of popup, it is able to use “PIN”.
  • - You can give the same title to any consecutive or subordinate popup.
Interaction
Specs
Properties
1-line PIN field consists of the following elements:

1. Box (Mandatory)
  • - Provide an input field for number entry.
  • - The first field should be focused on initially and then the field that a number is being entered into should be focused on.
2. Text (Mandatory)
  • - Provide the message for each case as below.
CASE MESSAGE
Default Enter your PIN. (The default PIN is <<A>>) (Interim terms)
Changed PIN Enter your PIN. (Interim terms)
Enter invalid PIN The entered PIN is incorrect. Please try again. (Interim terms)
