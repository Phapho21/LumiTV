Overview

COMPONENT

Component is a meaningful information and interaction unit, basic elements that does not split.

The property table is defined to guide component as below.

<< COMPONENT NAME >>
PROPERTY USE VALUE ADDITIONAL
Property 1 ◎ <<value>> Mandatory property
Property 2 ○ <<value>> Use property
Property 3 X Non-use property
FEATURE DESCRIPTION
Feature 1 <<value>>
PROPERTY
  • - It is elements that consist of each component.
  • - USE line shows whether property is used or not; ◎ = Mandatory property / ○ = Use / X = Non-use
  • - Mandatory property must be not deleted or edited because it is required element for any layout.
  • - Non-use property is marked as dimmed to recognize clearly.
  • - Property which contains certain component already includes its component name on the property table. If you don’t use that property, you can delete and write new value.
FEATURE
  • - This part shows component’s unique features.
  • - It can describe like Function, scroll direction, default sort order and so on.
  • - You can write value on DESCRIPTION column.
