C_BUTTONBASIC
Definition

Basic button is used the most, and fundamental button.

Usage
  • - Use this button as the first priority unless there are any other critical issues.
  • - Display the button label that describes the button action.
Button Label
  • - Use short, concise labels that sufficiently explain the action.
  • - Start labels with an imperative verb for buttons used to perform actions.
    Choose labels that contain part of or the entire function name or secondary window's title for buttons used to open another function or window.
  • - Avoid using generic labels, such as "Yes" or "OK". Choose specific labels that clearly describe the action of a button.
  • - Provide users with a button to cancel the selected actions. Use a standard Cancel button when there is no other button that allows users to cancel an action.
    In this case, place the Cancel button to the right or at the bottom of the screen.
  • - Do not provide a Cancel button when the action cannot be canceled.
  • - Use the OK button for a simple confirmation.
Button Order
  • - Place the buttons required to perform an action first and the buttons required to cancel the current task or close the window, such as the Cancel or Done button, last.
  • - Place the button required to perform a recommended action as the very first option.
  • - For "Yes" and "No" buttons, place the "Yes" button first. However, make the "No" button the focused button when it is the recommended action.
Interaction
  • - Use icon button if there is no space.
  • - Place button label on icon button’s tooltip when focus on.
Specs
Properties
Basic button consists of the following elements :
1. Text or Icon (Mandatory)
  • - The Icon Should clearly describe the button action.
  • - The text label alone clearly describe the button action.
2. Tooltip (Optional, if icon type, Mandatory)
  • - Provide a button label in a tooltip when the button is focused on.
  • - Display a tooltip if necessary to facilitate the user’s understanding, besides when focused on.
